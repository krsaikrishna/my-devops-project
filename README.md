# My DevOps Github Project (Task 4)

## Overview
This project demonstrates Git version control workflows using branching, pull requests, tags, and documentation.

## Branching Strategy
- **main** – Stable production-ready branch  
- **dev** – Integration/testing branch  
- **feature/** – Used for new features or updates

## Technologies Used
- Git
- GitHub

## How to Use
1. Clone the repo  
2. Create a feature branch  
3. Commit and push changes  
4. Raise a Pull Request to merge
